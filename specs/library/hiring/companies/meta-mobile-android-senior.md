# What We're Looking For

Instagram Android serves 2+ billion people, which means every line of code you write gets magnified by scale most engineers never see. A 1ms improvement in feed scroll saves literal years of human time globally. This isn't hyperbole — it's Tuesday.

We move fast and break things, but "break things" at our scale means different things. Your code runs on everything from flagship Galaxy phones to 3-year-old budget devices in emerging markets. You'll own features that ship to hundreds of millions of users within weeks, not months. Our internal tooling is probably better than your last startup's entire tech stack.

The bootcamp process means you'll spend your first 6 weeks shipping code across multiple teams before choosing Instagram. By the time you land here, you'll understand how Meta actually works — not the version you read about on Blind.

## Must-Haves

- Shipped Android apps to 10M+ users (Instagram scale problems are different)
- Deep understanding of Android performance profiling — Systrace, method tracing, memory analysis beyond basic leak detection
- Experience with cross-platform mobile architecture (we share significant business logic between iOS/Android)
- Has worked on feed/timeline products or similar infinite scroll experiences
- Can debug performance issues in production that only reproduce at scale

## Strong Signals

- Contributed to major Android open source projects (AOSP, major libraries)
- Has opinions about Kotlin vs Java that go beyond syntax preferences
- Experience with mobile CI/CD at scale (hundreds of engineers, thousands of commits/day)
- Built developer tools or internal frameworks that other engineers actually used
- Has worked through a major platform migration (think Java 8→11, or Gradle migration)

## Anti-Patterns

- Only worked on apps with <1M users (scaling patterns are fundamentally different)
- Can't articulate the performance implications of their architectural choices
- Treats mobile as "just frontend" without understanding platform constraints
- Has never been on-call for a mobile app (our stuff breaks, you'll need to fix it)
- Focuses on code elegance over user impact metrics

## Interview Focus

- System design for mobile at Instagram scale — how do you handle 100M users hitting a new feature simultaneously?
- Performance debugging war stories — what was the gnarliest performance issue you've solved and how did you find it?
- Technical leadership examples — how do you influence without authority when the Android team is 40+ engineers?
- Hackathon-style problem solving — we'll give you an ambiguous product problem and see how you break it down
- Cross-platform thinking — how would you design a feature that needs to work identically on iOS and Android?
