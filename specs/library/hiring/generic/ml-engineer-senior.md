# What We're Looking For

I need someone who's moved beyond Jupyter notebooks and actually deployed models that real users depend on. Too many "ML engineers" have only ever trained models on clean datasets with obvious signals. We need someone who's dealt with data drift, model decay, and the 90% of ML work that happens after the training script finishes.

Looking for someone who understands that ML engineering is fundamentally different from research. Research optimizes for novelty; production ML optimizes for reliability, maintainability, and business impact. The best candidates have war stories about models that looked great in development but failed spectacularly in production.

## Must-Haves

- Deployed ML models to production that served real traffic (not batch jobs or demos)
- Built or significantly improved ML infrastructure (training pipelines, serving infrastructure, monitoring)
- Experience with model versioning, A/B testing, and gradual rollouts in production
- Can code fluently in Python and at least one other language (Go, Scala, Java)
- Has dealt with concept drift and model retraining in production systems

## Strong Signals

- Built feature stores or ML platforms that other teams used
- Experience with both batch and real-time inference at scale
- Has strong opinions about MLOps that go beyond "we use MLflow"
- Can explain the tradeoffs between model complexity and operational overhead
- Experience mentoring ML engineers or transitioning software engineers to ML

## Anti-Patterns

- "Production experience" that was really just scheduled batch jobs
- Can't explain why they chose their specific model architecture beyond "it worked"
- Treats data quality as someone else's problem
- Only comfortable with one ML framework and can't adapt
- Dismisses traditional software engineering practices as irrelevant to ML

## Interview Focus

- Production failure stories: what went wrong, how they debugged it, what they changed
- Architecture decisions: why they built their ML systems the way they did
- Cross-functional collaboration: how they work with data engineers, product managers, and backend teams
- Technical leadership examples: projects where they influenced technical direction beyond their own code
