# What We're Looking For

I need someone who's moved beyond just following test scripts and can think like a user while building like an engineer. At this level, you should be able to take a feature spec and independently design a testing strategy that actually catches the bugs that matter.

Too many QA engineers get stuck in the "happy path" mindset. I want someone who naturally thinks about edge cases, error states, and the weird ways users actually interact with our product. You should have strong opinions about what's worth automating versus what needs human judgment, and be able to articulate why.

## Must-Haves

- 3+ years hands-on test automation (Selenium, Cypress, or similar)
- Can write maintainable test code, not just scripts that work once
- Experience with API testing beyond just checking 200 responses
- Has worked in an agile environment and understands when to push back on deadlines for quality
- Can triage bugs effectively — knows the difference between "this breaks checkout" and "this button is 2px off"

## Strong Signals

- Has experience with performance testing or load testing tools
- Comfortable with database queries for test data setup and validation
- Can pair with developers on testability improvements
- Has opinions about test pyramid and can explain their testing strategy
- Experience with mobile testing (iOS/Android) or cross-browser compatibility

## Anti-Patterns

- Only manual testing experience with no automation skills
- Treats every bug as equally critical
- Can't explain why they chose a particular testing approach
- Waits for "complete" requirements before starting test planning
- Views QA as separate from development rather than integrated

## Interview Focus

- Walk through how they'd approach testing a new feature from scratch
- Bug triage scenarios — how do they prioritize and communicate issues
- Test automation philosophy — what they automate vs. test manually and why
- How they've influenced product quality beyond just finding bugs
- Examples of working with developers to improve testability
