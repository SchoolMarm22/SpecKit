# What We're Looking For

I'm looking for someone early in their career who has built something real with Node.js — not just followed tutorials, but actually deployed code that users interact with. Could be a side project, bootcamp capstone, or internship work. What matters is that they've felt the pain of debugging async code at 2am and lived to tell the story.

The best junior engineers I've hired weren't necessarily the ones with the most GitHub stars or the fanciest side projects. They were the ones who could clearly explain their code, admit when they didn't know something, and showed genuine curiosity about how systems work under the hood. We have senior engineers who can teach the advanced stuff, but I can't teach someone to care about craft.

## Must-Haves

- Built and deployed at least one complete Node.js application (REST API or web app)
- Understands async/await vs callbacks vs promises — and more importantly, when each causes bugs
- Can write SQL queries beyond SELECT * (joins, basic indexing concepts)
- Has debugged a production issue, even if it was just on their own project

## Strong Signals

- Asks good questions about error handling and edge cases during technical discussions
- Has read other people's code and can explain what it does
- Shows curiosity about performance — even basic stuff like "why is this endpoint slow?"
- Can articulate why they chose Express vs Fastify vs Koa (shows they think about tradeoffs)

## Anti-Patterns

- Only knows Node.js through frameworks — can't explain what the event loop does
- Treats databases like magic boxes (never looked at query execution plans or thought about indexes)
- Copy-pastes code without understanding it, especially around authentication or security
- Thinks testing is "optional" or something only senior engineers do
- Can't explain the difference between 400 and 500 status codes

## Interview Focus

- Code walkthrough of their most complex project — focus on their debugging process
- Simple system design: "How would you build a URL shortener?" (not looking for perfection, but thought process)
- Basic Node.js concepts: event loop, streams, middleware patterns
- Growth mindset: what they're excited to learn, how they approach unfamiliar problems
