# What We're Looking For

I'm looking for someone early in their career who genuinely enjoys building user interfaces and has the curiosity to dig into how things work under the hood. You don't need 5 years of Vue experience, but you should have built something real with it — even if it's a side project or bootcamp capstone.

The best junior hires I've made were people who could explain their code clearly and showed they were actively learning. I'd rather hire someone who built a Vue todo app and can walk me through their component structure than someone who claims to know "all the frameworks" but can't explain why they made specific choices.

This role will involve working on our customer dashboard, so you'll be writing Vue components, connecting to REST APIs, and collaborating closely with our design team. You'll have a senior engineer as a mentor, but I need someone who can own small features end-to-end.

## Must-Haves

- Built at least one complete Vue 3 application (personal project counts)
- Comfortable with JavaScript fundamentals — can explain closures, promises, and the event loop
- Has written CSS that works across browsers without copying from Stack Overflow
- Can consume REST APIs and handle loading/error states thoughtfully
- Demonstrates curiosity about frontend performance (even if limited experience)

## Strong Signals

- Asks good questions during technical discussions
- Has opinions about component organization or state management
- Contributes to open source or maintains a technical blog
- Experience with Vue's Composition API or Pinia
- Has worked with designers or shows interest in design-engineering collaboration

## Anti-Patterns

- Claims expertise in 10+ frameworks but can't explain Vue reactivity
- Only ever copied code without understanding what it does
- Dismisses testing as "too hard" or "not my job"
- Can't articulate why they chose Vue over React for a project
- Expects to only work on "fun" features and avoid bug fixes

## Interview Focus

- Code walkthrough of their Vue project — focus on decision-making process
- How they approach learning new concepts or debugging issues
- Reaction to feedback on their code (growth mindset vs. defensiveness)
- Understanding of Vue's reactivity system and component lifecycle
- How they handle uncertainty or being stuck on a problem
