# What We're Looking For

I'm looking for someone who gets excited about mobile development and has built something real, even if it's small. The best junior Android engineers I've hired had a personal project in the Play Store — not because it was successful, but because they cared enough to see it through to publication.

We need someone who can grow into our existing codebase without needing their hand held for every task. Our Android app has real users and real complexity (offline sync, push notifications, camera integration), so while I don't expect you to architect these systems, I do expect you to be curious about how they work and willing to dig into unfamiliar code.

## Must-Haves

- Published at least one Android app (personal project counts)
- Comfortable with Kotlin and can explain why it's better than Java for Android
- Understands Activity and Fragment lifecycle without constantly looking it up
- Has debugged a real crash using Android Studio's tools
- Can implement a basic RecyclerView with custom ViewHolders

## Strong Signals

- Contributes to open source Android libraries or has meaningful GitHub activity
- Has opinions about architecture patterns (MVP vs MVVM vs MVI) even if they're still forming
- Asks good questions about existing code rather than just copying patterns
- Shows curiosity about Android internals (how does View rendering work, what's the difference between Parcelable and Serializable)

## Anti-Patterns

- Only experience is following tutorials without building anything original
- Can't explain the difference between a Service and an IntentService
- Dismisses testing as "too hard" or "not worth it" for mobile
- Treats Android development as "just Java with some XML"
- Never reads documentation, only Stack Overflow answers

## Interview Focus

- Walk through their published app: what was hard, what would they do differently
- Live coding a simple UI with basic interaction (not algorithm puzzles)
- How they approach unfamiliar APIs or Android concepts
- Their debugging process when something doesn't work as expected
