# What We're Looking For

I need someone who can take a Figma design and ship it without needing their hand held, but also someone who will ask the right questions before they start coding. Too many mid-level engineers either bikeshed every decision or bulldoze ahead without thinking through edge cases.

The sweet spot is someone who's made enough mistakes to have opinions, but hasn't become dogmatic about those opinions yet. They should be able to own a feature end-to-end while still being coachable when they hit something outside their experience.

## Must-Haves

- 2-4 years production React experience with hooks, not just class components converted to hooks
- Has debugged performance issues in React DevTools and can explain what caused them
- Comfortable with modern CSS (flexbox, grid) and responsive design principles
- Can work with REST APIs and handle async state management (Redux, Zustand, or similar)
- Has shipped features that real users actually use, not just internal tools

## Strong Signals

- Can articulate why they chose one state management solution over another
- Has experience with testing (Jest, React Testing Library) and writes tests that actually catch bugs
- Understands the difference between controlled and uncontrolled components
- Has worked with TypeScript or is excited to learn it

## Anti-Patterns

- Only knows Create React App and panics when they see a custom webpack config
- Writes components with 500+ lines and thinks that's normal
- Can't explain the difference between useMemo and useCallback
- Treats accessibility as something you "add later" 
- Has never worked on a team larger than 3 people

## Interview Focus

- Walk through a recent feature they built: decisions made, problems encountered, what they'd do differently
- Code review skills: can they spot issues in a React component beyond syntax errors?
- How they approach breaking down a complex UI into components
- Their debugging process when something isn't working
