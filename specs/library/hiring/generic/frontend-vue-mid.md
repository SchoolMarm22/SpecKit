# What We're Looking For

I need someone who can own features end-to-end without constant guidance. Not a senior who architects systems, but someone who can take a mockup and API spec and ship something users actually want to use.

The Vue ecosystem moves fast. I want someone who understands the difference between Vue 2 and Vue 3's reactivity systems, not because they memorized the docs, but because they've hit the edge cases in production. Someone who's written enough components to have opinions about when to use slots vs props vs provide/inject.

Most importantly, I need someone who cares about the user experience beyond just "making it work." Performance matters. Accessibility isn't an afterthought. Error states exist and should be designed, not just handled.

## Must-Haves

- 2+ years building production Vue applications (Vue 3 Composition API preferred)
- Has built and maintained a component library or design system
- Experience with Vue ecosystem tools (Pinia/Vuex, Vue Router, Vite/Webpack)
- Can write performant components and explain why reactivity choices matter
- Has shipped features that handle real error states and loading patterns

## Strong Signals

- Has migrated a Vue 2 app to Vue 3 (shows they understand breaking changes)
- Contributes to code reviews with thoughtful feedback about component design
- Experience with testing Vue components (Vue Test Utils, Vitest, or similar)
- Has worked with designers to implement complex interactions or animations
- Can explain when to use computed vs watch vs watchEffect

## Anti-Patterns

- Only knows Vue through tutorials or bootcamps without production experience
- Treats every state problem like it needs a global store
- Writes components that are either too generic or too specific
- Can't explain the difference between reactive and ref
- Has never worked on a Vue app that outlived the initial development phase

## Interview Focus

- Component design decisions: when they chose composition over inheritance
- State management philosophy: local vs global state, when to lift state up
- Performance debugging: how they've identified and fixed slow renders
- Testing strategy: what they test and what they don't, and why
- Real bug stories: complex issues they've debugged in Vue applications
