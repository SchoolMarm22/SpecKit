# What We're Looking For

At staff level, I need someone who shapes how we build frontend systems, not just what we build. You should have strong opinions about Vue architecture patterns, state management strategies, and how to scale frontend codebases across multiple teams without creating a maintenance nightmare.

This isn't about being the best Vue developer on the team—it's about being the person who makes everyone else better Vue developers. I'm looking for someone who's lived through the pain of large Vue codebases and has concrete strategies for avoiding common architectural pitfalls.

## Must-Haves

- 6+ years Vue experience including Vue 2 to Vue 3 migration at scale
- Designed and implemented component libraries or design systems used by multiple teams
- Led technical decisions that affected >5 engineers for >6 months
- Deep understanding of Vue's reactivity system, composition API patterns, and performance optimization strategies
- Experience with micro-frontend architectures or large monorepo frontend strategies

## Strong Signals

- Has strong opinions about Pinia vs Vuex vs composables for state management and can defend them with real examples
- Built or significantly contributed to Vue tooling, plugins, or open source ecosystem
- Can articulate the tradeoffs between different Vue testing strategies (unit vs integration vs e2e)
- Experience mentoring senior engineers, not just juniors
- Has written RFCs or technical specs that influenced team/org direction

## Anti-Patterns

- Only worked on Vue applications, never built reusable Vue libraries or frameworks
- Can't explain performance bottlenecks they've actually debugged and fixed in production
- Treats TypeScript as optional rather than essential for large Vue codebases
- Never had to maintain Vue code written by other teams
- Focuses on individual productivity over team/org multiplier effects

## Interview Focus

- System design: How would you structure a Vue application that 20 engineers work on?
- Technical leadership stories: Times they influenced technical direction beyond their immediate team
- Vue-specific deep dives: Reactivity edge cases, render function optimizations, SSR considerations
- Legacy migration strategies: How they'd approach migrating a large Vue 2 codebase to Vue 3
- Mentorship philosophy: How they've grown other senior engineers' technical skills
